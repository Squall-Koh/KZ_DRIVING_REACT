🗄️ Table Schema 설계

React Web & Flutter 공통 서버 DB 설계안입니다. track_data, expenses는 제거되었습니다.

PK Primary Key
FK Foreign Key
UQ Unique
NN Not Null
users
사용자/운전자 테이블
ColumnTypeConstraints설명
idVARCHAR/UUIDPK사용자 고유 ID
nameVARCHARNN운전자 이름
departmentVARCHAR소속 부서
roleVARCHAR권한 (admin, driver)
created_atTIMESTAMPNN생성 일시
updated_atTIMESTAMPNN수정 일시
vehicles
차량 테이블
ColumnTypeConstraints설명
idINTEGER/UUIDPK차량 고유 식별자
nameVARCHARNN차량 표시명
vinVARCHARUQ차대번호 (BLE 매핑)
license_plateVARCHAR차량 번호판
modelVARCHAR차량 모델
yearINTEGER연식
created_atTIMESTAMPNN생성 일시
updated_atTIMESTAMPNN수정 일시
ble_devices
BLE 단말기 테이블
ColumnTypeConstraints설명
idINTEGER/UUIDPK단말기 고유 식별자
mac_addressVARCHARUQ NN기기 MAC 주소
device_nameVARCHAR블루투스 기기명
vehicle_idINTEGER/UUIDFK vehicles.idBLE 설치 차량
statusVARCHARNNactive / inactive
created_atTIMESTAMPNN생성 일시
notifications 신규
알림 테이블
ColumnTypeConstraints설명
idINTEGER/UUIDPK알림 고유 식별자
user_idVARCHAR/UUIDFK users.id알림 수신자 ID
typeVARCHARNNsystem, maintenance 등
titleVARCHARNN알림 제목
messageTEXTNN알림 세부 내용
is_readBOOLEANDEFAULT FALSE읽음 여부
created_atTIMESTAMPNN발송 일시
work_days
출/퇴근 및 당일 운행기본 테이블
ColumnTypeConstraints설명
idINTEGER/UUIDPKWork day 고유 식별자
user_idVARCHAR/UUIDFK users.id운전자 ID
dateDATENN운행 근무일자
check_in_timeTIMESTAMPNN출근 시각
check_out_timeTIMESTAMP퇴근 시각
vehicle_idINTEGER/UUIDFK vehicles.id승차 차량 ID
odo_startINTEGER일일 시작 누적거리
odo_endINTEGER일일 종료 누적거리
created_atTIMESTAMPNN생성 일시
trips
개별 운행 기록 테이블
ColumnTypeConstraints설명
idINTEGER/UUIDPK운항건 고유 식별자
work_day_idINTEGER/UUIDFK work_days.id당일 출퇴근 ID
trip_numberINTEGERNN당일 N번째 운행
start_timeTIMESTAMPNN출발 시간
end_timeTIMESTAMP도착 시간
start_lat / lngDECIMAL출발지 위도/경도
end_lat / lngDECIMAL도착지 위도/경도
start_odoINTEGER출발 시점 누적거리
end_odoINTEGER도착 시점 누적거리
distance_kmDECIMAL해당 트립 운행 거리
created_atTIMESTAMPNN생성 일시
maintenance_records 신규
차량 정비 이력 테이블
ColumnTypeConstraints설명
idINTEGER/UUIDPK정비 고유 식별자
vehicle_idINTEGER/UUIDFK vehicles.id정비 대상 차량
service_dateDATENN점검/정비 일자
service_typeVARCHARNN정비 항목 (엔진오일, 타이어 등)
costINTEGER소요/결제 비용
center_nameVARCHAR정비소 이름
descriptionTEXT특이사항/상세 설명
created_atTIMESTAMPNN레코드 생성 일시
